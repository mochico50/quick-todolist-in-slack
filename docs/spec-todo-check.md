# 機能仕様書 - todo-check（既存Bot）

## 1. 概要

Slackの買い物チャンネルで「1買い物1メッセージ」の運用を支えるSlash Command Bot。  
直近2週間の未完了の買い物投稿を一覧表示する。

**実装:** Google Apps Script（`todo-check/app.gs`）  
**変更方針:** 原則変更しない。Gmail連携機能はgmail-watcherで別実装する。

---

## 2. 運用ルール

| 状態 | リアクション | 絵文字コード |
|------|------------|------------|
| 完了 | ✅ | `:white_check_mark:` |
| キャンセル | ❌ | `:x:` |
| 未完了 | （リアクションなし） | - |

- 1つの買い物につき1メッセージを投稿する
- 買ったら自分で ✅ をつける（手動）
- キャンセルは ❌ をつける（手動）
- gmail-watcherが注文検知した場合も ✅ をつける（自動）
- 相談はスレッドで行う

---

## 3. `/todo-check` コマンド仕様

### Given（前提）
- ユーザーが買い物チャンネルで `/todo-check` を実行する

### When（操作）
- Slackが `/todo-check` のPOSTリクエストをGASのWebアプリURLに送信する

### Then（結果）
- チャンネルに `📋 start checking todo list...` の親メッセージが投稿される
- 未完了投稿が1件以上ある場合：そのスレッドに各投稿へのパーマリンクが1件ずつ投稿される
- 未完了投稿が0件の場合：そのスレッドに `Todoなし🎉` が投稿される

---

## 4. 未完了投稿のフィルタ条件

直近14日以内のメッセージのうち、以下をすべて満たすものを「未完了」とする。

| 条件 | 内容 |
|------|------|
| テキストあり | `msg.text` が存在する |
| subtypeなし | `msg.subtype` が存在しない（チャンネル参加通知などを除外） |
| Bot投稿でない | `msg.bot_id` が存在しない |
| コマンドでない | テキストが `/` で始まらない |
| ✅がない | reactions に `white_check_mark` が含まれない |
| ❌がない | reactions に `x` が含まれない |

---

## 5. 必要なSlack APIスコープ

| スコープ | 用途 |
|---------|------|
| `channels:history` | メッセージ取得 |
| `chat:write` | メッセージ投稿 |
| `reactions:read` | リアクション取得 |

GASのスクリプトプロパティに設定する。
